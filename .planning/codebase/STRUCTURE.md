## Directory Purposes

**`optimizer/src/`:**
- Purpose: Core transformation and component extraction logic
- Contains: AST traversal, semantic analysis, code generation
- Key files: `transform.rs` (main logic), `js_lib_interface.rs` (public API)

**`optimizer/src/component/`:**
- Purpose: Component representation and metadata
- Contains: Component definitions, IDs, Qrl markers, source information
- Key files: `component.rs`, `id.rs`, `qrl.rs`, `language.rs`, `source_info.rs`

**`optimizer/src/ext/`:**
- Purpose: Extension traits for OXC AST types
- Contains: Helper methods for AST node construction
- Key files: `ast_builder_ext.rs`, `expression_ext.rs`

**`optimizer/src/test_input/`:**
- Purpose: Test input fixtures
- Contains: `.tsx` and `.js` files used by snapshot tests
- Examples: `test_example_1.tsx`, `test_example_2.tsx`, `test_project_1/src/`

**`optimizer/src/snapshots/`:**
- Purpose: Insta snapshot test outputs
- Contains: `.snap.yaml` files with expected transform results
- Key directory: `swc_expected/` for comparing against original SWC output

**`napi/src/`:**
- Purpose: Node.js binding layer
- Contains: NAPI function exports, async task management
- Key file: `lib.rs` (single module with `transform_modules` export)

## Naming Conventions

**Files:**
- Test inputs: `test_<name>.tsx` or `test_<name>.js` (e.g., `test_example_1.tsx`)
- Multi-file tests: `test_<project_name>/src/` directory structure
- Snapshots: `<test_name>.snap.yaml` (generated by insta)
- Modules: `<concept>.rs` (e.g., `transform.rs`, `segment.rs`)

**Directories:**
- Subdomain grouping: `component/`, `ext/` for related functionality
- Test data: `test_input/`, `snapshots/` (no camelCase)
- Workspace crates: lowercase with hyphens (`qwik-optimizer`, `qwik-optimizer-napi`)

**Types (Rust structs/enums):**
- PascalCase: `TransformGenerator`, `OptimizedApp`, `SegmentBuilder`
- Single word with suffix: `Language`, `Target`, `QrlType`

**Functions/Methods:**
- snake_case: `transform_modules()`, `from_source()`, `get_binding_identifier()`
- Constructors: `new()`, `from_*()`, `with_*()`

**Imports:**
- Grouped by type: OXC imports, stdlib, crate modules
- Path aliases used sparsely (mostly relative within crate)

## Where to Add New Code

**New Transform Pass/Feature:**
- Primary code: `optimizer/src/transform.rs` - Add visitor methods to `TransformGenerator`
- Supporting types: `optimizer/src/component/` - Add to appropriate component submodule
- Tests: `optimizer/src/test_input/test_<feature_name>.tsx` + `optimizer/src/js_lib_interface.rs` test function

**New Diagnostic/Error Type:**
- Error definition: `optimizer/src/error.rs` - Add variant to `Error` enum
- Processing failure: `optimizer/src/processing_failure.rs` - If non-fatal, add `ProcessingFailure` variant
- Conversion: `optimizer/src/js_lib_interface.rs:error_to_diagnostic()` - Add match arm

**New Component Attribute/Metadata:**
- Type definition: `optimizer/src/component/component.rs` - Add field to `QrlComponent`
- ID calculation: `optimizer/src/component/id.rs:calculate_hash()` - Include in hash if needed
- Serialization: `optimizer/src/js_lib_interface.rs:SegmentAnalysis` - Add field if exposed to JS

**New Entry Strategy:**
- Strategy enum: `optimizer/src/entry_strategy.rs` - Add `EntryStrategy` variant
- Policy implementation: `optimizer/src/entry_strategy.rs` - Implement `EntryPolicy` trait
- Integration: `optimizer/src/transform.rs` - Handle in strategy dispatch logic

**New Extension Trait:**
- File: `optimizer/src/ext/<concept>_ext.rs` - Create new file
- Module registration: `optimizer/src/ext/mod.rs` - Add `mod` and `pub(crate) use` declarations
- Usage: Import in `optimizer/src/transform.rs` where needed

**Test Input File:**
1. Create `optimizer/src/test_input/<test_name>.tsx` or `.js`
2. Add test function in `optimizer/src/js_lib_interface.rs` tests module:
   ```rust
   #[test]
   fn test_<name>() {
       assert_valid_transform!(EntryStrategy::Segment);
   }
   ```
3. Run `cargo test -p qwik-optimizer` then `cargo insta review` to generate snapshot

## Module Organization

**Public API (re-exported from lib.rs):**
- `component::*` - Component types (Language, SourceInfo, QrlComponent, Import, etc.)
- `js_lib_interface::*` - Types matching original Qwik optimizer (TransformModulesOptions, TransformOutput, etc.)
- `source::*` - Source loading (Source enum)
- `transform::*` - Core transformation (TransformOptions, OptimizationResult)
- `macros!` - Test macros (function_name, assert_valid_transform)

**Private/Internal (not re-exported):**
- `dead_code` - Dead code trait implementations
- `illegal_code` - Illegal code detection
- `import_clean_up` - Import reconciliation
- `entry_strategy` - Entry point policies
- `segment` - Segment naming
- `ref_counter` - Reference counting trait
- `error` - Error types
- `prelude` - Type aliases
- `ext` - AST extension traits
